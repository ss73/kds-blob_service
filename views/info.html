Blob Service

This service provides a means of storing binary data in the local file system. The files are stored in an application sub folder as JSON documents with Base64-encoded data and a name. The name needs to be unique, storing under an exisitng name results in the contents being overwritten.

Endpoints

/

This information page.

/store

Upload JSON or form data to the index. If the endpoint is accessed using GET, a form will be displayed which will post the following parameters:
  • name : A blob name
  • content : Base64 encoded binary data
Application access to the service should typically not use the form itself, but POST the JSON representation of the above as the body, for example:

{"name" : "A unique name", "content" : "QmFzZTY0IGVuY29kZWQgY29udGVudA=="}

/retrieve/:name

Returns a JSON representation of the stored blob. The :name parameter should be sent using the GET method. Omitting the parameter in a GET request lists all available blobs (i.e. /retrieve).
